# Code of conduct

Tebis should be a direct, respectful place to collaborate.

## Expected behavior

- Be clear and kind.
- Assume good intent, but accept correction.
- Keep criticism focused on the work.
- Respect different experience levels, tools, operating systems, and workflows.
- Avoid sarcasm or jokes at another person's expense.

## Unacceptable behavior

- Harassment, threats, or personal attacks
- Sexualized language or attention
- Doxing or sharing private information
- Repeatedly derailing issues or pull requests
- Dismissing security concerns without investigation

## Reporting

For conduct concerns, open a GitHub issue asking for a private maintainer
contact method. Do not include private details in the issue.

Security issues should not be reported through this process. Use
[SECURITY.md](SECURITY.md) instead.
